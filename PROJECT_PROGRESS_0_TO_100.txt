PROJECT GOAL
------------
Build one premium mobile-first platform serving:
- B2C: factory to personal/retail customer.
- B2B: factory to approved business/wholesaler.
- Owner/admin: catalogue, prices, photos, stock, applications, requests and website content without code editing.

CONFIRMED PRODUCT DIRECTION
---------------------------
Status: APPROVED

- Premium public landing page and searchable catalogue.
- Separate Personal/B2C and Business/B2B experiences.
- Phone-number and OTP authentication.
- Supabase Test OTP for development and staging.
- First-time customers choose Personal or Business only after OTP verification.
- Personal creates retail_customer access.
- Business creates wholesale_applicant access pending owner approval.
- Wholesale prices, MOQ and carton information remain hidden until approval.
- Admin is never shown as a public signup option.
- Pre-authorised owner/admin users use /admin/login.
- Owner controls product photos, retail/wholesale prices, stock and availability.
- Default transaction mode: ENQUIRY_ONLY / MANUAL_CONFIRMATION.
- No payment gateway in version one.
- Public UI must be product-led, authentic and conversion-focused.
- Admin UI remains light, calm and operational.

DEPLOYMENT FAILURE 001 — GENERATED SELF-SERVICE BINDING
-------------------------------------------------------
Observed:
- Cloudflare code 10143.
- Auto-generated WORKER_SELF_REFERENCE pointed to a Worker that did not exist.

Cause:
- No committed OpenNext/Wrangler configuration during first deploy.
- Wrangler auto-migration generated a self-service binding.

Fix:
- Added explicit wrangler.jsonc and open-next.config.ts.
- Removed self-service binding for first deployment.
- Added OpenNext/Wrangler dependencies.

DEPLOYMENT FAILURE 002 — RECURSIVE OPENNEXT BUILD
-------------------------------------------------
Observed:
- Deployment ran longer than 20 minutes.
- Repeated OpenNext build headings and repeated opennextjs-cloudflare build calls.

Cause:
- package build script pointed to OpenNext.
- OpenNext invoked package build and recursively invoked itself.

Fix:
- build restored to next build.
- cf:build added for OpenNext.
- Cloudflare Build command set to npm run cf:build.

DEPLOYMENT FAILURE 003 — WORKER NAME AND FUTURE DATE
----------------------------------------------------
Observed:
- Config name gowtham-fireworks did not match connected Worker gowthamfireworks.
- Cloudflare code 10021 because compatibility date 2026-07-24 was ahead of Cloudflare UTC.

Fix:
- Worker name changed to gowthamfireworks.
- Compatibility date changed to 2026-07-23.
