PROJECT GOAL
------------
Create a premium, mobile-first Gowtham Fireworks Industries catalogue and landing website with a secure owner-controlled admin panel.

The owner must be able to manage the website without editing code, including:
- Add and edit products.
- Upload product photographs.
- Set MRP and selling price.
- Set availability.
- Organise products by category.
- Feature selected products.
- Create combo or celebration packs.
- Receive and manage customer enquiries.
- Update business and website settings.

The customer-facing website will use a catalogue and enquiry workflow rather than a conventional online-payment checkout.

CONFIRMED PRODUCT DIRECTION
---------------------------
Status: APPROVED

1. Public premium fireworks landing page.
2. Searchable product catalogue.
3. Product details with photograph, price and availability.
4. Enquiry basket instead of direct online payment.
5. WhatsApp-assisted enquiry submission.
6. Secure owner admin page.
7. Owner enters each product photo, price and availability.
8. Separate Supabase project/account for Gowtham Fireworks.
9. Gowtham Fireworks data must not be mixed with CapDent, Astro or other projects.

PROPOSED TECHNICAL DIRECTION
----------------------------
Status: PROPOSED — confirm before implementation

- Frontend and admin: Next.js with TypeScript.
- Styling: Tailwind CSS or an equivalent maintainable design system.
- Database: Supabase PostgreSQL.
- Authentication: Supabase Auth for the owner/admin.
- Product images: Supabase Storage initially.
- Hosting and CDN: Cloudflare.
- Public catalogue and admin may share one application, with the admin under /admin.
- Database protection: Supabase Row Level Security.
