PROJECT GOAL
------------
Build one premium mobile-first platform serving:
- B2C: factory to personal/retail customer.
- B2B: factory to approved business/wholesaler.
- Owner/admin: catalogue, price, stock, application, request and website-content control without code editing.

CONFIRMED PRODUCT DIRECTION
---------------------------
Status: APPROVED

1. Premium public landing page and searchable catalogue.
2. Separate Personal/B2C and Business/B2B experiences in one application.
3. Phone-number and OTP authentication.
4. Supabase Test OTP during development and staging.
5. New customers choose Personal or Business only after OTP verification.
6. Personal accounts become retail_customer accounts.
7. Business accounts become wholesale_applicant accounts until owner approval.
8. Wholesale prices, MOQ and carton information remain hidden until approval.
9. Admin is never offered as a public signup option.
10. Pre-authorised admins use /admin/login and route directly to /admin after role verification.
11. Owner controls product photos, retail/wholesale prices, stock and availability.
12. Use a dedicated Gowtham Fireworks Supabase project only.
13. Default transaction mode is ENQUIRY_ONLY / MANUAL_CONFIRMATION.
14. No payment gateway in version one.
15. Visual direction: “Sivakasi Midnight Festival”.
16. Public UI is celebratory; admin UI is light, calm and operational.

DEPLOYMENT FAILURE 001
----------------------
Date: 23 July 2026
Status: ROOT CAUSE FOUND AND CONFIG FIX COMMITTED

Observed failure:
- Cloudflare error code: 10143.
- Generated service binding WORKER_SELF_REFERENCE referenced Worker gowtham-fireworks.
- The Worker did not yet exist, so the first deployment was rejected.

Cause:
- Repository had no committed wrangler.jsonc or open-next.config.ts.
- Running npx wrangler deploy in CI triggered automatic Next.js/OpenNext migration.
- The migration generated R2 incremental-cache resources and a self-service binding.
- Cloudflare requires a service-binding target Worker to exist before deploying a Worker that references it.

Fix committed:
- Added explicit wrangler.jsonc without a self-service binding.
- Added explicit open-next.config.ts.
- Added @opennextjs/cloudflare and wrangler as project devDependencies.
- Changed npm run build to generate .open-next output.
- Preserved existing mandatory deployment command npx wrangler deploy.
- Added .open-next and .dev.vars exclusions to .gitignore.
- Added docs/CLOUDFLARE_DEPLOYMENT.md.

Fix commits:
- package/OpenNext scripts: b76c9b98a1e4a0afd254052ae8e66eb6790e51d2
